02 CLI 命令参考

linkshell start

启动桥接会话。自动启动内置网关，桥接 AI 代理进程并生成配对二维码。

--daemon      后台守护模式运行
--provider    指定 AI 提供者 (claude / codex / custom)
--screen      启用远程桌面共享
--port        指定网关端口 (默认 8787)
linkshell gateway

单独启动网关服务器。适用于独立部署网关到远程服务器的场景。

--port        指定监听端口
--host        指定绑定地址
--daemon      后台守护模式运行
linkshell setup

交互式初始化配置。引导你完成网关地址、AI 提供者等基础设置。

无额外参数，交互式引导完成配置
linkshell doctor

环境诊断工具。检查 Node.js 版本、依赖完整性、网络连通性等。

自动检测并报告所有环境问题
linkshell status

查看当前运行状态。显示活跃会话、网关连接状态和守护进程信息。

显示会话 ID、连接状态、运行时长
linkshell stop

停止守护进程。优雅关闭桥接会话和内置网关。

安全断开所有连接并清理资源

03 部署方式

本地直连（最简单）

CLI 内置网关，同一局域网内手机直连电脑。零配置，开箱即用。

$
npm i -g linkshell-cli
linkshell start --daemon

远程服务器

将网关部署到公网服务器，配合 Nginx 反向代理 + HTTPS，跨网络访问。

# 服务器端
linkshell gateway --daemon
# 本地
linkshell start \
  --gateway wss://your.server

Docker 部署

使用 Docker Compose 一键部署网关。最低要求 1 核 512MB 内存。

docker compose up -d

05 常见问题

需要公网服务器吗？

不需要。CLI 内置网关，同一局域网内手机可以直连电脑。如果需要跨网络访问，才需要将网关部署到公网服务器。

支持哪些 AI 代理？

目前支持 Claude Code、OpenAI Codex，以及任意自定义 Shell 进程。通过 --provider 参数指定。

手机 App 在哪里下载？

iOS 版本通过 TestFlight 分发，Android 版本可从 GitHub Releases 下载 APK。正式版本上架 App Store 和 Google Play 后会在此更新。

数据安全吗？会经过第三方服务器吗？

本地直连模式下数据完全在局域网内传输。远程模式下数据经过你自己部署的网关中继，不经过任何第三方服务器。网关不存储任何终端数据。

远程桌面功能如何工作？

使用 --screen 参数启动后，通过 WebRTC 以 30fps 实时传输桌面画面。网络条件不佳时自动降级为截图流模式，确保始终可用。
